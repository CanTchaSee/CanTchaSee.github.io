RSA explained

Posted Aug 30 2020-08-30T21:12:00+01:00 by cantchasee

So you are interested in RSA :) in this post I’ll try to explain how and why RSA work

What is RSA?

Before we start our why? and how? we should ask first what is RSA ? basically RSA is an asymmetric cryptographic algorithm wich mean that you have a key let’s name it key1, you encrypt your message with key1 but you can’t recover it using the same key, you’ll need another key let’s name it key2 to decrypt the message and yes

\( key1 \ne key2 \)

These keys are named public key and private key

That’s why RSA is different you can encrypt your data using a key and decrypt it using another one because it’s an asymmetric cryptographic algorithm unlike symmetric cryptographic algorithm like (AES,…) that use one password

As we stated before RSA use two keys one for encryption and the other for decryption precisely we use public key to encrypt message and private key to decrypt ciphertext and yes the name of the keys mean you can give your public key to anyone there is no problem you can print it in you clothes and wear it there is no problem but no one should know your private key because as the name tell it’s private :P

Note: There is no problem in encrypting data using private key we can decrypt it using public key ;)

Why RSA?

Ok now you should be wondering why using RSA and not just a symmetric encryption like AES the following picture is a simple and easy to understand usage of it:

  • Everyone know alice and bob public keys
  • No one know alice and bob private keys this also mean that bob and alice don’t know each other keys
  • Alice encrypt her message using her private key to get cipher1
  • Alice encrypt cipher1 using bob public key witch everyone know it to get cipher2
  • Alice send cipher2 to Bob
  • Even if someone got cipher2 he can’t decrypt it because he doesn’t know bob private key
  • Bob decrypt cipher2 using his private key to get cipher1
  • Bob decrypt cipher1 using Alice public key witch available to everyone to get Alice message

You can’t do this using symmetric encryption ;)

But there is a problem RSA is slooow compared to something like AES so we can’t rely on it to send data and we can’t use AES without giving the encryption/decryption key to someone by doing so we will risk the key being stolen by someone so the idea here is:

  • Send AES key securely using RSA
  • Now that both person have the AES key they can start sharing encrypted data using the fast AES without the need of RSA

And this is why we need RSA, Ok we know what is RSA why we need it and now how RSA work?

The Mathematics of RSA

ok let’s start

  1. we need two prime numbers \(p,q\)
  2. we calculate \(n = p.q\)
  3. we calculate \(L = \Phi(n)\) we call it Euler Totient
  4. we calculate \(e\) such that \(1 < e < L\) and \(gcd(e,L) = 1\)
  5. we calculate \(d\) such that \(d.e \equiv 1 [L]\) And now we have everything to construct our public and private keys:
    • Public Key: \((n,e)\)
    • Private Key: \((n,d)\)

And now let’s say a message \(m < n-1\) and the ciphertext \(c < n-1\) to get encrypt the message we have:

\(c \equiv m^e [n]\)

and to decrypt it we have:

\(m \equiv c^d [n]\)

This is possible because:

\(m \equiv {m^e}^d [n]\)

Let’s prove that last one:

  1. Using Euler theorem we get:
    • \(m^{\Phi(n)} \equiv 1[n] \Rightarrow m^{k.\Phi(n) + 1} \equiv m[n]\)
  2. Using Bezout theorem we get:
    • \(e.d - k.L = 1\)
  3. This mean \(k.\Phi(n) +1 = k.L + 1 = e.d\)
  4. We got \(m^{e.d} \equiv m[n]\)
  5. we put \(c = m^e\) this mean \(m \equiv c^d [n]\)
  6. since \(c < n-1\) using the same step that we got we will have \(c^{d.e} \equiv c [n]\) and we know that \({c^d} \equiv m [n]\) then we got \(m^e \equiv c [n]\)

And here is it we got RSA hope you enjoyed it :D
